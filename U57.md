make all compilerar classerna till bytecode 
make run kör bytecoden till classen Calculator och därmed huvudprogrammet
make test compilerar test classerna och kör dem sedan

Ett byggverktyg är ett verktyg som låter programeraren kompilera och länka kod på de valda sätten utan att behöva skriva in det i input var gång utan istället skrivs korta kommandon. Ett fel som byggverktyg hhjälper undvika är att den i c kan kontrollera om specifika delar behövs omkompileras innan en del körs dvs om den delen som du vill köra är depending av en ouppdaterad del. 

Båda makefilerna använder sig av liknande namn bakom de funktionerna som kallas då det är bekvämt att ha make test som ett sätt att köra tester även om vägen till att testerna faktiskt körs varierar kraftigt.

Just make är betydligt kraftfullare i C då den smidigt sköter de specificerade kraven (dependencys) och då C gör länkning i samband  med kompilering fungerar detta mycket bättre där än i Java som ofta JIT compileras. Samt så har C ofta en plattare fil struktur då Java organiseras baserat på klasser och underklasser vilket gör det komplicerat med sökvägar.
